# PC 관리자를 전상표로 바꾸고 OneDrive 전상표로 사용하는 방법

이 PC의 **Windows 로그인 계정**을 전상표로 바꾸고, **OneDrive도 전상표 계정**으로 사용하는 절차입니다.

---

## 전제 조건

- **전상표**로 사용할 **Microsoft 계정**이 있어야 합니다.  
  (Outlook.com, live.com, hotmail.com 등으로 가입한 계정 또는 Microsoft에 연결된 이메일)
- 현재 PC에 **전유미(xnsemfk@gmail.com)** 로 로그인된 상태에서 진행합니다.

---

## 방법 1: 전상표를 새 Windows 사용자로 추가 후, 그 계정으로 로그인해 사용 (권장)

이 방법은 기존 전유미 계정을 지우지 않고, **전상표 계정을 추가**한 뒤 **평소에는 전상표로 로그인**해서 쓰는 방식입니다.

### 1단계: 전상표 Microsoft 계정을 이 PC에 추가

1. **Windows 설정** 열기  
   `Win + I` 또는 **시작 메뉴 → 설정**
2. **계정** → **가족 및 다른 사용자** (또는 **다른 사용자**)
3. **다른 사용자 추가** (또는 **이 PC에 다른 사용자 추가**) 클릭
4. **Microsoft 계정이 없음**이면 → **Microsoft 계정으로 사용자 추가** 선택 후, 전상표의 **Microsoft 계정 이메일**과 **비밀번호** 입력
5. 사용자 추가가 끝나면, 목록에 전상표 계정이 나타남
6. **관리자로 만들기**  
   - 전상표 계정 이름 클릭 → **계정 종류 변경** → **관리자** 선택 → 확인

### 2단계: 전상표로 로그인

1. **시작 메뉴** → 현재 사용자(전유미) 아이콘 클릭
2. **전상표** 계정 선택 후 로그인 (비밀번호 입력)
3. 첫 로그인 시 Windows가 전상표용 바탕화면·폴더를 만듦  
   → 이제 **이 PC의 기본 사용자를 전상표로 쓰는 것**과 같음

### 3단계: 전상표 계정에서 OneDrive 설정

1. **전상표로 로그인한 상태**에서
2. 작업 표시줄 **OneDrive(구름) 아이콘** 클릭  
   - 이미 전유미나 다른 계정으로 연결되어 있으면:
     - OneDrive 아이콘 → **설정(톱니바퀴)** → **계정** → **이 PC 연결 해제**
     - 해제 후 다시 OneDrive 실행 → **전상표 Microsoft 계정**으로 로그인
3. 전상표 계정으로 로그인하면 **전상표 OneDrive(상표 - 개인 등)** 가 이 PC 사용자와 일치
4. 필요 시: **동기화 및 백업** → **내 중요한 Windows 폴더 백업 관리** 에서  
   데스크톱/문서/사진을 전상표 OneDrive로 백업할지 선택

### 정리

- **평소에는 전상표로 로그인**해서 사용
- OneDrive는 **전상표 로그인 상태**에서만 전상표 계정으로 연결하면 됨
- 전유미 계정은 PC에 그대로 있어서, 필요하면 다시 전유미로 로그인 가능

---

## 방법 2: 현재 전유미 계정을 전상표 Microsoft 계정으로 “전환”

**지금 쓰는 로그인 계정(전유미) 자체를** 전상표 Microsoft 계정으로 바꾸는 방법입니다.  
기존 바탕화면·문서 등이 **전상표 계정 폴더로 넘어갈 수** 있으므로, 중요한 자료는 미리 백업해 두세요.

### 1단계: Microsoft 계정 연결 확인

1. **설정** → **계정** → **이메일 및 계정**
2. 현재 **Microsoft 계정**이 전유미(xnsemfk@gmail.com)로 되어 있음

### 2단계: 로컬 계정으로 전환 후, 전상표 계정으로 다시 연결

1. **설정** → **계정** → **이메일 및 계정**
2. **Microsoft 계정으로 로그인** 항목에서 **로컬 계정으로 전환**  
   → 비밀번호 입력 후, 로컬 계정용 비밀번호 새로 설정
3. 전환이 끝나면 **로컬 계정**으로 로그인된 상태
4. 다시 **설정** → **계정** → **이메일 및 계정**
5. **Microsoft 계정 추가** 또는 **Microsoft 계정으로 로그인** 선택
6. **전상표의 Microsoft 계정** 이메일·비밀번호 입력
7. 이제 이 PC의 Windows 로그인 계정이 **전상표 Microsoft 계정**으로 바뀜

### 3단계: OneDrive

- 같은 사용자(전상표)로 로그인되어 있으므로, OneDrive 앱에서 **전상표 계정**으로 연결하면 됨  
  (이미 다른 계정이 연결돼 있으면 **이 PC 연결 해제** 후 전상표로 다시 로그인)

---

## 방법 1 vs 방법 2

| 구분 | 방법 1 (새 사용자 추가) | 방법 2 (계정 전환) |
|------|-------------------------|----------------------|
| 전유미 계정 | PC에 그대로 유지, 필요 시 로그인 가능 | 이 PC에서는 로컬 계정만 남거나 제거됨 |
| 기존 파일(바탕화면·문서 등) | 전유미 폴더에 그대로 있음 | 전상표 폴더로 옮기거나 다시 정리해야 할 수 있음 |
| 난이도 | 비교적 단순 | 계정 전환 시 실수 시 복구가 번거로울 수 있음 |

**일반적으로는 방법 1**을 추천합니다. 전상표를 “이 PC의 주 사용자”로 쓰면서, 전유미 계정은 그대로 두는 방식입니다.

## 전상표로 바꾼 뒤 SPK Mobile Bot 프로젝트

- 프로젝트가 **현재 전유미 사용자 폴더**  
  `C:\Users\user\.cursor\spk  mobile  bot\spk-mobile-bot`  
  에 있으면, **전상표로 로그인한 뒤**에는 경로가  
  `C:\Users\전상표계정이름\.cursor\spk  mobile  bot\spk-mobile-bot`  
  같은 형태가 **아닙니다**.
- **전상표 사용자**로 로그인하면 **전상표 전용 사용자 폴더**가 생기므로,  
  기존 전유미 폴더 안의 프로젝트는 **전상표 로그인 상태에서는 보이지 않을 수 있습니다**.
- **그래서:**
  - **방법 1**을 쓸 경우:  
    SPK Mobile Bot을 쓰실 때는 **전유미로 로그인**해서 쓰거나,  
    전상표로 쓰려면 프로젝트 폴더를 **전상표 사용자 폴더**로 복사/이동하거나,  
    공통 위치(예: `C:\Dev\spk-mobile-bot`)에 두고 두 계정 모두 접근하게 할 수 있습니다.
  - **방법 2**를 쓸 경우:  
    전상표 계정 전환 전에 프로젝트를 **다른 드라이브나 폴더로 복사**해 두었다가,  
    전환 후 전상표 사용자 폴더(예: `C:\Users\전상표\.cursor\...`)로 옮기거나,  
    GitHub에서 `git clone`으로 다시 받을 수 있습니다.

## 요약

1. **PC “관리자”를 전상표로 쓴다** = **Windows 로그인 계정을 전상표로 쓴다**고 보면 됩니다.
2. **권장:** **방법 1** — 전상표를 **다른 사용자로 추가** → **관리자 권한 부여** → **평소에 전상표로 로그인**.
3. 전상표로 로그인한 뒤 **OneDrive** 앱에서 **이 PC 연결 해제** 후 **전상표 Microsoft 계정**으로 다시 로그인하면, 이 PC에서 OneDrive 전상표(상표 - 개인) 사용이 됩니다.
4. SPK Mobile Bot은 **어느 사용자로 주로 작업할지** 정한 뒤, 해당 사용자 폴더나 공통 경로에 두고 사용하시면 됩니다.
